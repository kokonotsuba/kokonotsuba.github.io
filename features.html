Features

Disclaimer

This doesn't cover every feature right now. But functions as an overview of what you can expect.

Basic

Kokonotsuba has all basic features you expect from an Imageboard. Such as:

  • Sage
  • Deleting posts
  • Tripcodes
  • Quoting text
  • Quote links

Neat

Global message

The Global message is a instance-wide message that the Administrator can set. It appears near the post form

Blotter

Functions as a little news section. Which will display text along with the date the entry was added. A preview can be found underneath the post form, [Show all] leads to a page that displays all entries.

Entires can be created and deleted from the [Manage blotter] page. Which an Administrator can find on the link bar in [Admin]

Banners and Ads

Kokonotsuba has a banner and ad (internally called 'fullbanner')

Overboard

The Overboard is a page that will show all non-hidden boards on a single page, it can be accessed from [Overboard] at the top right of the board view. Boards can also be hidden by the user with the Filter boards form. Which will not show any threads from boards that arent check-marked.

Accounts

Kokonotsuba has staff accounts. Staff can change their password and view their own Action record (a counter of how many bans, deletions, etc.) from [Accounts] in [Admin]

More on account management in the next section.

Admin/mod tools

A lot of BBSes suffer from poorly written mod tools and features that make moderating the site a pain. Kokonotsuba however, has some superior implementations.

JS post deletion

If you have JS enabled on your browser. When deleting a post, attachment, or muting, some javascript will hide or modify the post's html so the page doesn't have to be fully refreshed. Which can make dealing with spam attacks a breeze.

For Janitors the post will simply disapear and a message will confirm if the post was deleted or not.

For Mods and above, the post will have its opacity reduced and be hidden from regular users. More on this next.

Trash bin / Deleted posts

On Kokonotsuba, deleting things doesn't automatically remove them from the system right away. Deleting a post will make it hidden from users and Janitors. This is extremely useful for preventing rogue Janitors or compromrised mod accounts from permanently wiping the board.

At [Admin], if your role is high enough, you'll see [Manage deleted posts] on the link bar. On this page you'll see all deleted posts along with who deleted them, when, and optionally whyin the form of a note.

From the live view as a Moderator or above, you can see deleted posts/threads/attachments with reduced opacity and [DELETED] on them - these posts are only visible to you.

You can restore any of these posts by clicking the [Restore] button.

For illegal content you'll want it permanently removed from the server - which you can achieve by clicking the [Purge] button. Which will wipe the post and attachment from the database and filesystem.

Banning and Mutes

Account management

Accounts can be created, deleted, and promoted/demoted by the Adminstrator

Board management

Rebuild board / Manage rebuild

Rebuild board will just rebuild

Autosage

Threads can be autosaged, also known bump lock, which prevents the thread from bumped even the reply is a non-sage.

Sticky

Pins a thread to the top of the board - above all other threads. The thread will however, be only showed in its regular bump from the overboard.
